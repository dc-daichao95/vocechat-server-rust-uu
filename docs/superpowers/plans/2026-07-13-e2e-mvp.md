# E2E Server+Web MVP Implementation Plan

> **For agentic workers:** implement task-by-task; checkboxes track progress.

**Goal:** Ship Server E2E plumbing (keys, opaque messages, session flags, webhook redact) and Web DM text encrypt/decrypt MVP (`e2e_ver=1` X25519 + AES-GCM), without REALITY in-process.

**Architecture:** Server stores only public keys and opaque ciphertext; clients encrypt before send. Web uses Web Crypto. Full Double Ratchet deferred; `e2e_ver=1` is static ECDH session key derived from identity keys (documented as interim).

**Tech Stack:** Rust/Poem/sqlx, React/WebCrypto
