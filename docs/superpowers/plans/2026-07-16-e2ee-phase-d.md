# E2EE Phase D (in progress) — Client binding

Date: 2026-07-16

## Done in this slice

### Core (`voce-e2ee-core`)

- `RatchetStateDto` persistence + FFI methods:
  - `ratchet_init_alice` / `ratchet_init_bob`
  - `ratchet_encrypt` / `ratchet_decrypt`
  - `dm_session_open_initiator` / `dm_session_open_responder`
- Envelope v2 `DR+AES-GCM` supports optional `x3dh_initial`
- FFI round-trip test `ffi_dm_session_roundtrip`

### Flutter (`vocechat-client-uu`)

- `lib/services/e2e_v2_core.dart` — FFI to `voce_e2ee_core.dll`
- `lib/services/e2e_v2_identity.dart` — generate + store + publish v2 identity/SPK (reuse SPK)
- `lib/services/e2e_v2_dm.dart` — DM Double Ratchet encrypt/decrypt + session store
- Send path prefers v2 when peer has signed prekey; else v1
- Decrypt path handles `DR+AES-GCM` via `E2eCrypto.rewriteDetailInPlace`
- Bootstrap publishes v1 (if protocol&lt;2) then v2 so `/bundle` prefers SPK device

### Web (`vocechat-web-uu`)

- `public/e2e-core/*` — WASM package (rebuilt with ratchet APIs)
- `src/app/e2e/v2_core.ts` / `v2_dm.ts` — WASM + DM session helpers
- `useSendMessage` — DM v2 encrypt when peer supports SPK
- `E2eText` — decrypt v2 envelopes (own outbound via `local_id` plaintext cache)
- Bootstrap publishes v1 then v2 (same as Flutter)

## Not yet (next slices)

- Channel Sender Keys via core
- Safety-number UI / block-on-key-change
- Device-link QR screens
- Multi-device fan-out for DR (currently one peer device session)
- Own-outbound history without `local_id` cache (cross-device)

## Enable v2 cutover

Admin sets login config `e2e_protocol_ver = 2` after clients with this build are deployed.
Until then, clients opportunistically use DR when both sides have v2 identity/SPK.
